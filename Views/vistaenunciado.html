    • Revista Electrónica jd&sc

    Universidad de Caldas

    Este sistema de software servirá para administrar una nueva revista electrónica para el Departamento de Ingeniería de Sistemas de la Universidad Caldas. La idea es que el sistema maximice la productividad del editor de la revista, brindándole un conjunto de herramientas que automaticen la evaluación de los artículos y su eventual publicación.
    El sistema debe permitir al editor de la revista la comunicación y gestión con un grupo de pares evaluadores y autores para publicar artículos en una página web de acceso público. El software facilitará la comunicación entre editor de la revista los pares y autores. Los lectores de la revista accederán a la página Web para buscar y descargar los artículos que les interesen de manera libre y gratuita. Los autores enviarán sus artículos mediante el sistema al editor de la revista utilizando un enlace “Enviar Artículo” que activará la aplicación. Es posible que este proceso sea ejecutado más de una vez si es solicitada una reescritura o corrección del artículo con base en las evaluaciones de los pares y la decisión del editor. Si el artículo es aceptado, el autor recibirá una notificación.
    Igualmente, si el artículo es rechazado, recibirá una notificación con una copia de las evaluaciones de los pares. La última opción es que el articulo sea condicionado a unos cambios sugeridos por los evaluadores, en cuyo caso, las evaluaciones serán enviadas junto con instrucciones para enviar el artículo revisado.
    Los pares evaluadores recibirán los artículos en una opción del sistema que dice listar artículos. Una vez evaluado el artículo se debe activar un enlace para enviar las sugerencias o correcciones del artículo al editor de la revista.
    El editor de la revista es quien tiene más interés en el sistema y, por consiguiente, quien más funcionalidad obtiene del software. Por un lado necesita funcionalidad básica para adicionar, modificar y borrar los datos de autores, pares, evaluaciones y artículos. Se podrá solicitar una modificación si el editor de la revista detecta que la información archivada del autor ha cambiado. La información que se requiere del autor es: su nombre completo, su dirección, su teléfono, su correo electrónico y su afiliación (Departamento, Institución.) Para los pares evaluadores además de la información del autor se pide estudios realizados como especializaciones, maestrías y postgrados.
    Con respecto a los artículos, el autor ingresa al sistema con su usuario y clave y los adiciona. Esta adición incluye un identificador(autonumerico), el título, tema(s), palabra(s) clave(s) y resumen del artículo, junto con un enlace para subir el documento. Los artículos como tal no se archivan en la base de datos, estos son guardados por el sistema en el disco duro del servidor en una carpeta llamada artículos con un nombre que equivale al identificador asignado en el sistema. Al entrar los datos de un artículo, se le debe asignar un estado que inicialmente es “recibido”.
    Posteriormente, el editor de la revista accede al sistema y puede visualizar los diferentes artículos y selecciona cada uno de estos y accede a la función de seleccionar par, que despliega la lista de pares evaluadores y donde debe seleccionar tres por artículo. En otro momento, el editor de la revista puede visualizar la evaluación que realizo cada uno de los pares(donde el sistema realiza un promedio de calificación y si es mayor a 4 es aprobado si es mayor a 3 menor a 4 pendiente con cambios y menor a 3 no aprobado). Una vez el artículo se pueda visualizar en las evaluaciones para a revisado.
    En cualquier momento el editor de la revista puede revisar el estado de los artículos registrados en el sistema. los que tengan estado “recibido” puede asignarles evaluador. A los que tengan estado revisado puede visualizar la decisión de los pares (“aceptado”, “rechazado”, “pendiente”). En cualquiera de los tres casos, la decisión activa un vínculo que permite enviar la notificación al autor y le permite también enviar los comentarios que realizaron cada uno de los pares.
    Además por si hay cualquier duda que tenga el editor de la revista de la revisión de un artículo el sistema debe tener un chat entre los pares evaluadores y el editor para comentar las dudas entre ellos.
    Si el artículo está en estado aceptado, el editor puede decidir publicarlo en la página web del sistema que es de acceso público.

    Algunas condiciones para la construcción de este software son:

    • La base de datos deberá estar implementada en una base de datos de libre distribución o en MySQL o postgreSQL.

    • herramientas adicionales: Esto implica que el acceso a la base de datos para la aplicación (cuya interface será vía Web para todos los usuarios, incluido el editor) debe hacerse con un mecanismo que permita usar lenguajes de programación de libre distribución (ejemplo php, Ruby, Phyton).

    • Utilizar en todo el sistema Css y boostrap para las vistas.

    • Jquery para que el sistema trabaje de forma asíncrona.

    • El diseño de la aplicación debe ser agradable y amigable, utilizar colores, tipos de letra, tamaño de letra, etc.

    • Se recomienda Node js para la construcción del chat(básico).

    • Se debe trabajar bajo patrón MVC.

    • Utilizar además tecnologías como DHTML, JavaScript y Prototype en toda la aplicación.
